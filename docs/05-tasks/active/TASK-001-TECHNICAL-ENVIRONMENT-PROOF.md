# TASK-001 — Доказательная инвентаризация технической среды

**Статус:** ACTIVE
**Дата создания:** 2026-07-06
**Владелец решения:** ChatGPT — архитектор и руководитель проекта
**Исполнитель:** Codex / CLI — только точное выполнение этой задачи
**Режим:** proof_only
**Модуль:** Platform & Contracts / cross-cutting operations
**Репозиторий:** `MaksimUnimax/AVITO_Mayak`
**Локальный путь:** `/opt/avito-mayak`

---

## 1. Цель

Получить доказанный, воспроизводимый и безопасный снимок технической среды, в которой будет разрабатываться и позднее запускаться проект «Маяк Авито».

Результат этой задачи нужен ChatGPT для принятия решений о техническом baseline. Задача не выбирает стек, не создаёт код, не создаёт сервисы и не меняет конфигурацию сервера.

---

## 2. Что необходимо доказать

1. Состояние Git-репозитория и удалённой ветки `main`.
2. Операционная система, архитектура, доступный объём CPU, памяти и диска.
3. Наличие и версии доступных инструментов разработки и выполнения.
4. Наличие и доступность container runtime.
5. Наличие локальных инструментов PostgreSQL, Redis и SQLite.
6. Наличие только перечисленных системных сервисов, без чтения секретов или конфигураций.
7. Список открытых TCP-портов без PID, process arguments, environment variables или иных секретов.
8. Что задача не изменила ни репозиторий, ни сервер.

---

## 3. Разрешённый scope

Разрешены только read-only команды, перечисленные в literal prompt этой задачи.

Разрешено создать один файл задачи:

`docs/05-tasks/active/TASK-001-TECHNICAL-ENVIRONMENT-PROOF.md`

Файл создаётся только из точного текста, переданного ChatGPT, и не интерпретируется исполнителем.

---

## 4. Запрещённый scope

Запрещено:

- создавать код, тесты, контракты, fixtures, Dockerfile, CI, workflow, базу данных, очередь, сервис, пользователя, ключ или секрет;
- устанавливать, удалять или обновлять пакеты;
- запускать, останавливать, перезапускать, включать или отключать системные сервисы;
- подключаться к базе данных, Redis или чужим контейнерам;
- читать `.env`, credentials, systemd unit contents, shell history, process arguments, environment variables, ключи или секреты;
- менять Git/SSH configuration;
- создавать архитектурные или продуктовые решения;
- вносить любые изменения кроме создания этого файла задачи и его commit/push.

---

## 5. Критерии приёмки

Задача считается выполненной, только если:

- файл задачи совпадает с переданным ChatGPT текстом побайтно;
- в `main` создан ровно один commit с этим единственным файлом;
- remote `main` указывает на этот commit;
- все доказательства среды получены read-only командами;
- в конце рабочее дерево чистое;
- отчёт не содержит секретов;
- ChatGPT, а не исполнитель, определяет следующий шаг.

---

## 6. Формат отчёта

Исполнитель возвращает только поля, указанные в literal prompt. Никаких рекомендаций, выбора стека, планов, архитектуры или следующего шага исполнитель не добавляет.
