# Маяк Авито — манифест документации

**Версия манифеста:** 1.0  
**Статус:** APPROVED для Documentation Bootstrap  
**Назначение:** компактная карта источников истины и обязательного порядка чтения.

## Быстрый вход в проект

При новом стартовом prompt с ссылкой на этот репозиторий ChatGPT обязан читать документы в порядке:

1. `README.md`
2. `docs/00-governance/PROJECT_ENTRYPOINT.md`
3. `docs/00-governance/CURRENT_STATE.md`
4. `docs/00-governance/ROADMAP.md`
5. последние релевантные записи `docs/00-governance/DECISION_LOG_APPEND_ONLY.md`
6. последние релевантные записи `docs/00-governance/WORKLOG_APPEND_ONLY.md`
7. `docs/00-governance/OPEN_DECISIONS.md`
8. нужный модульный playbook и активная задача, когда они будут созданы и прямо относятся к запросу.

Целевую модель и архитектурную карту читать целиком, когда требуется изменить границы продукта, модульную архитектуру или принять техническое решение, которое может влиять более чем на один модуль.

## Статусы документации

- `DRAFT` — рабочий документ, ещё не утверждён владельцем/архитектором как обязательный.
- `APPROVED` — обязательный действующий источник истины в своей области.
- `CANDIDATE` — подготовленная версия, которую нельзя использовать для production-решения до review.
- `SUPERSEDED` — заменённый документ; не удаляется, но больше не определяет правила.
- `ARCHIVED` — исторический документ, сохранённый для доказательств.

Правила изменений: `docs/00-governance/DOCUMENTATION_GOVERNANCE.md`.
